Method

How the atlas is curated

A concise workflow for keeping entries structured, comparable, and traceable.

Curation workflow

From paper to catalogue entry

1

Find

Candidate entries come from papers, repositories, project pages, benchmarks, and submitted suggestions.

2

Check

The entry is checked against the original paper, official repository, model card, project page, or weight release.

3

Record

Key fields are recorded: model name, scope, modalities, architecture, paradigm, tasks, access, and source links.

4

Update

Reviewed entries are added to the structured catalogue, which also updates the interactive landscape.

What is included

EO foundation models, benchmarks, datasets, embedding products, agents, and vision-language systems.

What is recorded

Scope, modality, sensors, architecture, modelling paradigm, downstream tasks, access, and links.

Links

Paper, code, weights, and project links point to original sources whenever available.

Tasks

Tasks are kept as specific labels, such as land-cover mapping, change detection, segmentation, VQA, retrieval, or regression.

Landscape

The default map uses curated plotting fields: modality-complexity tier and reported downstream evaluations.

Submissions

Submitted entries start as candidates and are checked before becoming public catalogue entries.

Complete entry

A stable row should include

  • Model or benchmark name
  • Original paper link
  • Modalities and sensors
  • Architecture and paradigm
  • Downstream tasks
  • Code, weight, and project links
  • Access and caveats
